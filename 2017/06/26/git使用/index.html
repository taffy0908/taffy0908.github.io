    • git使用

    使用git之前，先初始化一下设置：
    先设置一下用户名和email：

    • git config –global user.name “你的英文名”
    • git config –global user.email “你的邮箱”
    • git config –global color.ui true
    • git config –global push.default simple

    git连接外网的设置： > git config –global http.proxy http://10.12.16.16:808

    git服务器地址中，大家可以将git.sinosoft.com换成10.12.6.73，或者在C:\Windows\System32\Drivers\etc\hosts文件中加入 10.12.6.73 git.sinosoft.com一行。

    • git push
      • git push -u presandbox fa_tusi_back 向远端推送并关联本地分支
      • git push origin :issue1 删除远程的issue1分支
    • git reset –hard ba19ee1ec4797e360fe8374c5c5885c55b2f4c49 将当前分支回退到ba19ee1ec4797e360fe8374c5c5885c55b2f4c49
    • git commit -m “closes #33”提交并关闭issue33
    • git diff 9ba1de2cd90e84142724018ce1c086fe31100ea8^ – project/cms-mix/1.x/js/live/page.js 查看某次提交修复了什么文件
    • git status
      • git status -sb查看当前分支关联，并显示关联的远程分支
    • git branch
      • git branch -v 查看各个分支最后一次commit 信息
      • git branch –merge 查看哪些分支已被并入当前分支
        • 一般来说，列表中没有* 的分支通常都可以用git branch -d 来删掉。原因很简单，既然已经把它们所包含的工作整合到了其他分支，删掉也不会损失什么。
      • git branch –no-merged 查看尚未合并的工作
      • git branch -d branch-name 删除分支名称为branch-name的分支
      • git branch -r 查看远程分支
      • git branch -a | grep nitpick 过滤分支
      • git branch -rm 旧分支名 新分支名 分支改名
    • git cherry-pick xxxxx 合并某一次提交到当前分支
    • git checkout
      • git checkout -b [name] 创建新分支并立即切换到新分支
      • git checkout presandbox/fa_nitpick -b fa_nitpick 基于远程分支去创建本地分支，且关联，并切换到新分支
        • git checkout origin/master -b fa_aaa
        • git checkout -b test 67668d5基于某次提交创建新分支
      • git checkout baike – project/baike/1.x/css/common/right 检出bike分支下相关内容到本分支，便于参考
    • git remote 远程仓库相关命令
      • git remote -v 查看远程仓库
      • git remote add [name] [url] 添加远程仓库
      • git remote rm [name] 删除远程仓库
      • git pull [remoteName] [localBranchName] 拉取远程仓库
      • git push [remoteName] [localBranchName] 推送远程仓库
      • git remote set-url –push [remote-name] [newUrl] 修改远程仓库
        • 比如git remote set-url –push origin git@ued.dajie.com:ued/presandbox.git
      • 备注
        • 如果想把本地的某个分支test提交到远程仓库，并作为远程仓库的master分支，或者作为另外一个名叫test的分支，如下：
        • git push origin test:master // 提交本地test分支作为远程的master分支
        • git push origin test:test // 提交本地test分支作为远程的test分支
    • 重命名本地分支 git branch -rm 旧分支名 新分支名
    • 从某个tag检出分支 git checkout -b branch_name tag_name
    • push模式（必须添加）
      • git config –global push.default simple
    • git方法简写
      • git config –global alias.ci commit
      • git config –global alias.co checkout
      • git config –global alias.br branch
      • git config –global alias.st status
      • git config –global alias.dom “diff origin/master –name-only” #查看当前分支基于主干改了那些文件
      • git config –global alias.pom “pull origin master” #拉取主干代码更新当前分支
      • git config –global alias.sst “status -sb”
